## 이벤트 핸들러 등록 및 제거
1. 이벤트 핸들러 어트리뷰트 (구식)
> HTML 태그에 `onclick` 같은 어트리뷰트를 직접 작성하는 옛날 방식 HTML과 JavaScript 코드가 섞여서 유지보수가 어렵기 때문에, 현재는 사용하지 않는 것이 좋음
2. 이벤트 핸들러 프로퍼티 (과도기)
> 요소 객체의 `onclick` 프로퍼티에 함수를 직접 할당하는 방식 HTML과 JS를 분리했다는 점에서는 발전했지만, 하나의 이벤트에 오직 하나의 임무만 부여할 수 있다는 치명적인 단점
3. `addEventListener` (현대적인 표준)
>`element.addEventListener('이벤트종류', 실행할함수)` 형태로 사용하는, 가장 강력하고 유연한 현대적인 표준 방식
btn4.addEventListener('click', mission);

## 이벤트 핸들러 제거하기
> 이벤트를 제거할 때는 '어떤 임무'를 제거할지 정확히 알려줘야 하고, 제거하려는 함수는 반드시 이름이 있거나 변수에 담겨있어야 함
btn4.removeEventListener('click', mission);

## 이벤트 전파(Propagation)와 위임(Delegation)
> 하위 요소에서 발생한 이벤트는, 그 파문이 부모 요소를 따라 최상위까지 올라가는 '버블링' 현상 따라서 부모 요소에 달아둔 이벤트 리스너는, 자식 요소에서 발생한 이벤트에도 '반응'
이벤트 위임: 똑똑하게 일 처리하기 이벤트 버블링을 활용하면, 여러 자식 요소에 각각 이벤트를 달 필요 없이, 부모 요소에 이벤트 핸들러를 단 하나만 등록하여 모든 자식의 이벤트를 처리 이것을 '이벤트 위임'이라고 하며, 코드가 훨씬 효율적이고 유연해짐

## 이벤트 제어: preventDefault와 stopPropagation
1. `preventDefault`: 브라우저의 '자동 반응' 멈추기
폼(form)을 제출하면 페이지가 새로고침되는 것이 브라우저의 '기본 동작'
`preventDefault`를 사용하면, 이 기본 동작을 막고 우리가 원하는 작업을 먼저 수행할 수 있음 (예: 사용자가 입력한 값의 유효성 검사)
2. `stopPropagation`: 이벤트의 '연쇄 보고(버블링)' 멈추기
모달 창의 회색 배경을 클릭하면 창이 닫혀야 함
하지만 모달 창의 흰색 내용 부분을 클릭했을 때는 닫히면 안 되기에,
이때 'stopPropagation'을 사용해, 이벤트가 부모(회색 배경)로 전파되는 것을 막음

## 마우스와 키보드 이벤트
1. 마우스 이벤트
> 사용자의 마우스 움직임에 반응함 `click`(클릭), `dblclick`(더블클릭), `mousedown`(누를때), `mouseup`(뗄때), `mousemove`(움직일때) 등
mouseBtn.addEventListener('mouseup', (e) => {
    $mouseArea.innerHTML += 'click 이벤트 발생 <br/>';
   });
mouseBtn.addEventListener('mousedown', (e) => {
    $mouseArea.innerHTML += `mousedown, button=${e.button}<br/>`;
   });
mouseBtn.addEventListener('contextmenu', (e) => {
    e.preventDefault; // 브라우저 기본 메뉴가 뜨는 것을 막는다
    $mouseArea.innerHTML += `contextmenu 이벤트 발생(기본 메뉴 차단)<br/>`;
   });

2. 키보드 이벤트
> 사용자의 키보드 입력에 반응
    - keydown: 키를 누르는 순간 (누르고 있는 동안 계속 발생)
    - keyup: 키에서 손을 떼는 순간
    - event.key: 입력된 문자를 알려줌 (ex: 'a', 'A', 'Enter')
    - event.code: 실제 물리적인 키 코드를 알려줌 (ex: 'KeyA', 'Enter')
