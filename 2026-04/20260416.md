## 클래스형 컴포넌트
> 컴포넌트란? UI를 재사용 가능한 '부품'으로 나눈 것(예: 버튼 / 헤더 / 프로필 카드 등) React 엘리먼트를 반환하는 '자바스크립트 함수' 반드시 '대문자'로 시작해야 함
※ class는 className으로, onclick은 onClick으로 (카멜케이스)방식을 사용해야함

1. 앞 글자는 반드시 대문자로 시작
class TitleClass extends React.Component {
  render() {
    return React.Component (
      'h1', {className: "classComponent"},
      '클래스형 컴포넌트입니다.'
    )
  }
}

2. 함수형 컴포넌트
function OldWayComponent() {
  return React.createElement (
    'div',
    null,
    React.createElement('h1', {className: "greeting"}, 'createElement 방식'),
    React.createElement('p', null, '코드가 길고 복잡')
  );
}

3. JSX 방식
function NewWayComponent() {
  return (
    <div>
      <h1 className="greeting">JSX 방식</h1>
      <p>코드가 간단함</p>
    </div>
  );
}

## 컴포넌트에 대해서
1. {} 안에 자바스크립트 표현식을 넣을 수 있음
2. HTML 속성과 약간 다른 규칙을 따름
3. 인라인 스타일은 '객체'형태로 전달해야 함
4. 모든 태그는 반드시 닫혀야 함

## Props
> 부모 컴포넌트가 자식 컴포넌트에게 데이터를 전달하는 '통로' 읽기 전용(read-only) 데이터(자식은 Props를 수정할 수 없음) 데이터는 항상 위에서 아래로 흐름
※ 부모가 Props를 전달하지 않았을 경우를 대비한 '기본값'을 설정할 수 있음

## propTypes
> 각 Prop이 어떤 '타입'이여야 하는지, '필수'인지 아닌지를 검사 규칙이 맞지 않으면 브라우저 콘솔에 경고를 표시하여 실수를 방지 
Greeting.propTypes = {
  name: PropTypes.string,
  favoriteNumber: propTypes.number
}
function App() {
  const name1 = '홍길동';
  const name2 = '유관순';
  return (
    <Greeting name={123} favoriteNumber={3}>
        React 공부 중이예요
    </Greeting>
  )
}
ReactDOM.createRoot(document.getElementById('root')).render(<App/>);
