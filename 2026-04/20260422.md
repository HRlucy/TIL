## Next js
> 기본적으로 SSR 위주의 풀스택 웹 프레임워크를 주력으로 성장하고 있으며,풀스택 프레임워크답게 API 기능 및 서버 컴포넌트를 통한 서버측 기능과, React 컴포넌트를 통한 클라이언트 기능으로 나뉘어 통합 제공하는 프레임워크이다. 현재 node.js 기반의 안정적인 풀스택 프레임워크로 사랑받고 있으며, SSG(Static Site Generation) 방식 또한 제공한다.  (React의 대표적인 프레임워크)
HTML Parsing → CSS Parsing → Render → Layout → Paint
(프로젝트 시작: npx create-next-app@latest)

## 메소드 관련
1. usePathname() 현재 URL 경로를 읽어옴 -> 활성 탭/메뉴 스타일 적용 시
2. useParam() [폴더명] 형태의 동적 라우팅 파라미터를 읽어옴 -> 상세페이지(/menu/1)
3. useSearchParams() URL 끝의 쿼리스트링(?key=value)를 읽어옴-> 검색 결과 페이지, 필터링
4. useRoute() 함수/이벤트 내에서 프로그래밍 방식으로 페이지 이동-> 폼 제출 완료페이지 이동,뒤로가기
5. 'use client' 클라이언트 컴포넌트에서만 사용 가능(use XXX 등 react 문법을 사용할 때 처리해야 함)
6. filter 배열에서 콜백함수가 true인 요소로만 배열 반환

## 컴포넌트
> 엘리먼트를 반환하는 개념을 가진 화면의 구성 단위 컴포넌트에는 클래스형 컴포넌트와 함수형 컴포넌트가 있다.
- BrowserRouter: 라우팅이 필요한 컴포넌트들을 감싸는 컴포넌트
- Routes: Routes들을 묶어주는 단위(컴포넌트)
- Route: url 요청 주소와 컴포넌트를 매핑해 주는 단위(컴포넌트)

## JSX
> JSX는 React에서 UI를 구성하는 데 사용하는 자바스크립트의 확장 문법 HTML과 유사한 문법을 사용하여 UI를 작성할 수 있게 하며 직관적이고 쉽게 코드를 작성할 수 있으며 Babel과 같은 도구를 통해 JavaScript코드로 변환된다.

## Navigation Bar 
> 하이퍼텍스트 링크를 포함하고 있는 영역으로 고정적으로 편리하게 다른 HTML 페이지를 활용하기 위한 영역이다. React에서는 header와 footer 컴포넌트를 활용한 Layout 컴포넌트를 통해 페이지 구성을 만들고 Outlet이라는 컴포넌트가 라우팅에 의해 리렌더링 되는 방식으로 작성이 된다.

## Params
> url 요청을 하며 값을 조회할 때는 조회의 대상, 또는 조회에 필요한 데이터를 전달하게 된다.이 때 url 요청 시 같이 전달되는 값을 parameter라고 하며 크게 두 가지 방식으로 전달할 수 있다. 두 가지 방식은 react-router-dom에서 제공하는 useSearchParams와 useNavigate를 사용하여 url에 넘어온 parameter값을 추출할 수 있다.
- api: API 요청 관련 메소드들이 존재 하는 패키지
- components: 일반적인 컴포넌트가 들어가게 할 패키지
- data: data.json 파일이 존재 할 패키지
- layouts: Layout 컴포넌트가 존재 할 패키지
- pages: 라우팅에 따라 바뀌 페이지 역할을 할 컴포넌트가 존재 할 패키지
const onClickHandler = () => {
  // console.log(searchValue);
  navigate(`/menu/search?menuName=${searchValue}`);
};
