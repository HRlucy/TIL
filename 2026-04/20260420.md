## useEffect, useRef, useMemo , useCallback, promise 의 차이
1. useState
> state : 계속해서 변화하는 특정 상태, 상태에 따라 각각 다른 동작을 함 컴포넌트에서 변경가능한 데이터상태를 관리할 때 사용
ex) const [count, setCount] = useState(0);

2. useEffect
> effect : 메모리 누수를 방지하고 리소스를 확보, API 호출 데이터 불러오기 등의 비동기 작업을 수행할 때 사용, DOM 조작이나 스크롤 위치 설정을 하고 싶을 때 사용 컴포넌트가 렌더링될 때마다 특정 작업을 수행하도록 설정
ex) useEffect(() => {
  return () => {

  };
}, [] '의존성배열');

3. useRef
> DOM 요소 또는 컴포넌트 인스턴스를 참조하는 데 사용되는 훅입니다. 이 훅을 사용하면 DOM 요소에 직접 접근, 인스턴스 변수를 생성할 수 있음 useRef에 저장된 값은 아무리 변해도 컴포넌트가 리렌더링되지 않음
ex) const scrollPosition = useRef(0);

4. useMemo
> 계산 비용이 높은 함수의 결과 값을 기억하고, 동일한 인자로 재실행될 때 이전에 계산한 값을 반환(재사용)하여 성능을 최적화하는 역할
ex) const value = useMemo(() => {
  return result;
}, [dependency]);

5. useCallback
> 함수 재생성을 방지하고 렌더링 성능을 최적화하는 데 도움을 줌 컴포넌트의 상태가 프롭스가 변경될 때 컴포넌트가 다시 렌더링되어 의존성이 변경될 때만 새로 생성되도록 함
ex) const onSubmit = useCallback(() => {
  return result;
}, [dependency]);

6. 커스텀 훅
> 이름이 'use'로 시작하는 일반 자바스크립트 함수 내부에서 useState와 useEffect를 마음껏 사용할 수 있음
ex) function useDelayTimer(delay) {
}

7. promise
> 비동기 작업이 미래의 언젠가 끝날 텐데, 작업이 끝나면 성공(resolve)했는지 실패(reject)했는지 꼭 알려줄게라고 약속하는 '약속 증서' 객체를 먼저 반환하는 개념
ex) const promise = new Promise((resolve, reject) => {

})

## useState / useRef의 차이
# 값 변경에 대한 리렌더링
1. useState
> 상태가 변경되면 컴포넌트가 리렌더링 
2. useRef
> 상태가 변경되도 컴포넌트의 리렌더링을 트리거 하지 않음

# 값의 변경과 기록
1. useState
> 새로운 값을 기반으로 새로운 상태를 생성하고 컴포넌트를 다시 렌더링
2. useRef
> 값이 변경되더라도 컴포넌트를 리렌더링하지 않음

# 렌더링에 영향을 주지 않는 값
1. useState
> 상태값의 변경은 컴포넌트의 리렌더링을 유발
2. useRef
> 값은 변경되어도 컴포넌트의 리렌더링에 영향을 주지 않음
