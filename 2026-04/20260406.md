## 형상관리(Configuration Management)란?
> 소프트웨어 버전 관리 툴이라고 한다.
소스의 변화를 끊임없이 관리하는 것을 말한다.
소스를 버전 별로 관리할 수 있어 개발할 때 실수로 소스를 삭제하거나, 수정하기 이전으로 돌아가야하는 경우 유용하게 사용.
누가 어떤 파일을 수정하였는지 알 수 있기에 병합하거나 수정된 소스를 추적하는 곳에 사용한다.

## 깃/SVN의 차이
* **SVN**:
ㆍ대부분의 기능을 완성하고 중앙 저장소에 commit
ㆍ개발자가 자신만의 버전을 가질 수 없음

* **GIT**:
ㆍ매우 빠른 속도를 사용하는 분산 저장소, svn보다 많은 기능을 지원하는 대신, bash 커맨드로 파일을 올리고 내려야 함
ㆍ개발자가 자신만의 버전을가질 수 있음(개발자와 서버의 저장소 독립적 관리 가능)
ㆍ마음대로 commit(push) 하다가 마지막에 push로 모든 파일을 보내는 것이 가능함
ㆍ브랜치(Branch) 생성과 병합이 SVN에 비해 압도적으로 빠르고 가볍다

- SVN : 중앙 서버에 소스코드와 히스토리를 저장
- Git : 소스코드를 여러 개발 PC와 저장소에 분산해서 저장

## GIT 작업 관련 분류 영역
작업 디렉토리(Working Directory)
스테이징 영역(Staging Area)
저장소(Repository)

## GIT 명령어에 관하여
* 로컬 사용자 등록
ㆍgit config --global user.name XXXX
ㆍgit config --global user.email XXXX

ㆍgit init 디렉토리 선언
ㆍgit status 상태값 조회
ㆍgit add 파일을 스테이징 영역에 추가
  - git add abcd.txt 파일 지정 추가
  - git add . 전체 파일 추가
ㆍgit commit 파일을 디렉토리에 커밋
  - git commit -m "" 커밋 시 코멘트를 추가
ㆍgit log 현재 로그를 확인
ㆍgit revert 현재 버전을 취소함
ㆍgit remote 현재 파일을 저장소에 연동함
ㆍgit branch XXXX 현재 저장소의 브랜치를 생성함
ㆍgit switch XXXX 현재 디렉토리의 연동을 스위칭함
ㆍgit switch -c XXXX 브랜치를 만들어 해당 브랜치로 스위칭함
ㆍgit merge XXXX 원격지에 저장되어있는 파일과 내 소스를 병합함
ㆍgit push 형상 관리저장소에 내 디렉토리의 파일을 올림
ㆍgit push origin XXXX 다른 깃 허브 저장소에 내 수정파일을 올림
ㆍgit pull 형상 관리저장소에서 파일을 받아옴
ㆍgit remote set-url upstream XXXXX 깃허브관리자의 깃허브주소에서 모든 파일을 받아옴
