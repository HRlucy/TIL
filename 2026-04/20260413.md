) class Student {

## 화살표 함수
>  ES6에서 도입된 함수를 간결하게 표현하는 문법 단순히 문법만 짧아진 것이 아니라, 'this'를 다루는 방식도 바뀌었음

1. 기본 변환 function
const message = function() {return 'hello world!';};
const arrowMsg1 = () => {return 'hello world!';};
console.log(arrowMsg1());

2. 매개변수가 하나일 때
const power = function(x) {return x * x;};
const arrowPower = (x) => x * x;

3. 매개변수가 없거나, 두 개 이상이면 소괄호() 필수
const greet = () => 'hello';
const sum = (a, b) => a + b;
console.log(arrowPower(3));
console.log(sum(1,5));

## 일반함수
> 일반 함수의 this는 호출할 때 결정됨 즉, 누가 부르냐에 따라 바뀜 하지만 화살표 함수는 자신만의 'this'를 가지지 않고, 바깥 'this'를 그대로 씀

1. 일반 함수를 사용한 경우
this.titles.forEach(function(title) {
      console.log(`${this.store} : ${title}`);
    })

2. 화살표 함수를 사용한 경우 
this.titles.forEach(title => console.log(`${this.store} : ${title}`));

## '순회 가능한' 데이터 컬렉션을 위한 통일된 규약(프로토콜)
> 배열, 문자열, NodeList, Map, Set등은 모두 '이터러블' 이터러블의 가장 큰 특징은 'for...of' 반복문을 사용할 수 있다는 것
> 이터러블 : '순회 가능한' 규약을 따르는 객제(for...of 가능) 유사 배열 : 인덱스와 length프로퍼티가 있어서 배열처럼 '생긴 객체'

## Rest 파라미터와 Spread 문법
> 똑같이 저 세개를 사용하지만, 쓰이는 위치에 따라 정반대의 역할을 함
- rest parameter: 여러 개의 값을 하나의 배열로 '모으기'
- spread syntax: 하나의 배열을 여러 개의 값으로 '펼치기'

## rest parameter
> 함수의 '매개변수' 자리에 사용하여, 정해지지 않은 개수의 인수들을 하나의 '배열'로 모아서 받음

## [spread syntax]
> 함수의 '인수'자리나, 배열/객체 리터럴 안에서 사용하여 배열의 요소들을 개별 값이 목록으로 펼쳐줌
