## Javascript

## while 반복문
> 조건이 참인 동안 코드를 반복 실행
while (조건식) {
  반복실행할 코드
}

## do-while 반복문
> 최소 한 번 실행한 후 조건을 검사
do {
  // 조건에 상관없이 처음 실행할 구문
} while (조건)

## continue / break 문
- continue: 현재 반복문의 나머지 코드를 건너뛰고, 다음 반복으로 넘어감
- break: 현재 반복문을 완전히 종료

예시)
// continue
for (let i=1; i<=10; i++) {
  if (i % 2 === 0) {
    continue;
  }
}

// break
for (let i=1; i<=10; i++) {
  if (i > 5) {
    break;
  }
}

## 객체
> 자바스크립트는 객체 기반 프로그래밍 언어로 원시 값을 제외한 나머지(함수, 배열, 정규 표현식 등)는 모두 객체
객체는 0개 이상의 프로퍼티로 구성된 집합이며 프로퍼티는 키(key)와 값(value)로 구성됨
자바스크립트에서 사용할 수 있는 모든 값은 프로퍼티 값이 될 수 있음
프로퍼티 값이 함수일 경우 method라 부름
- 프로퍼티 키가 식별자 네이밍 규칙을 준수하지 않는 이름일 경우 반드시 대괄호 표기법을 사용 console.log(obj["dash-key"]);

## 객체 순회 메소드
> forin / foreach 
- for (const key in dog) {
  console.log(`키: ${key}`);
  console.log(`값: ${dog[key]}`);
}
dog 값 안의 key를 출력하여 사용함

## 함수 호이스팅
> 함수 선언문은 런타임 이전에 자바스크립트 엔진에 의해 먼저 실행됨
따라서 함수 선언문 이전에 함수를 참조하거나 호출할 수 있음
함수 선언문이 코드의 선두로 끌어 올려진 것처럼 동작하는 자바스크립트 고유의 특징을 함수 호이스팅이라 함
함수 표현식은 런타임에 할당되기 때문에 반드시 함수 표현식 이후에 참조 또는 호출해야 함

## 함수: 일급 객체
> 자바스크립트에서 함수는 '값(value)'으로 취급될 수 있다
1. 변수에 담을 수 있음(함수 표현식)
2. 객체의 속성으로 담을 수 있음(메소드)
3. 함수의 인자(argument)로 전달할 수 있음
4. 함수의 결과(return value)로 반환될 수 있음

## 콜백함수
> 다른 함수의 '재료(인자)로 전달되어, 그 함수의 실행 시점을 제어하는 함수
- 예시) function calculator(calculatorCallback, a, b) {
  console.log('계산을 시작합니다.');
  const result = calculatorCallback(a, b);
  return result;
}

- 예시2) numbers.sort((a, b) => a - b);

## 즉시 실행 함수(IIFE)
> 함수를 정의하자마자 딱 한 번만 실행되는 '일회용 함수' 목적: 전역 오염 방지 + 변수 보호
- 예시)
(function() {
  let secretMessage = '이 변수는 이 안에서만 존재한다'; // 함수 안에서만 존재
  console.log('즉시 실행 함수가 실행되었습니다.');
  console.log(secretMessage);
})();

## 재귀 함수
> 함수가 자기 자신을 다시 호출하는 방식 반복되는 작업을 위해 사용하며, 반드시 '종료조건'이 있어야 함
- 예시)
function factorial(n) {
    if (n <=1) {
      return 1;
  }
  // 리턴이 없으면 RangeError: Maximum call stack size exceeded
  return n * factorial(n - 1);
}

## var키워드의 문제점
> ES5까지 사용되던 var키워드는 세 가지 주요 문제점을 가지고 있음
1. 변수 중복 선언 허용
2. 함수레벨 스코프
3. 변수 호이스팅 -> 가독성 및 예측 가능성 저하

## let / const 키워드
> var의 단점을 보완하여 나온 키워드
- var의 특성
1. 변수 중복 선언 금지
2. 블록 레벨 스코프
3. 변수 호이스팅이 일어나지 않는 것처럼 동작

- const의 특성
1. 선언과 동시에 반드시 초기화해야 함
2. 재할당 금지됨
3. 다른 객체로 변경하는 것은 불가 (※ 객체 내부의 프로퍼티를 변경하는 것은 가능)

)
* 1. 기본적으로 모든 변수는 const로 선언함(가장 안전)
 * 2. 값이 반드시 바뀌어야 하는 변수에만 let을 사용
 * 3. var는 사용하지 않음
